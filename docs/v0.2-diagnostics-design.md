# v0.2 — Diagnostics / semantic validation (design)

Status: approved design (2026-06-24), pending spec review → implementation plan.
Target: v0.2.0.

## Goal

In-editor diagnostics for `.kicad_dru` files: surface invalid rules as squiggles
(Problems panel + inline) as the user types.

## Governing principle — no false positives

KiCad's own *Check Rule Syntax* button remains the source of truth for whether a
rule is valid. The extension must therefore **only flag what is definitely
wrong; when unsure, stay silent.** A false squiggle on a rule KiCad would accept
is worse than no squiggle at all. Every design choice below serves this.

## Scope

### Tier A — structural (severity: Error)
Definite, catalogue-independent errors:
- Unbalanced parentheses (extra `)` or unclosed `(` at end of file).
- Unterminated string literal on a line.
- Missing or duplicate `(version N)` header, or a non-integer version.
- Unknown constraint type after `(constraint …)` (not in the catalogue).
- Unknown top-level / rule-body form keyword (not `rule`/`version` at top, or
  `constraint`/`condition`/`layer`/`severity` in a rule body).
- Invalid enum value: bad `(severity …)`, `(constraint disallow …)` category, or
  `(constraint zone_connection …)` value.
- Unit suffix on a unit-less constraint value (e.g. `(constraint track_angle
  (min 90mm))` — `track_angle`/`via_count`/`min_resolved_spokes`/`via_dangling`/
  `bridged_mask` take no unit).

### Tier B — expression / vocabulary (severity: Warning)
Catalogue-based; warnings (not errors) because the catalogue can lag a KiCad
release — a warning says "I don't recognise this," not "KiCad will reject it":
- Unknown property after `A.`/`B.`/`AB.` inside a condition/assertion string.
- Unknown function call.
- Unknown bare layer token in a `(layer …)` clause.
- Use of a KiCad-11 / master-only token (the net-chain family) — informational
  warning that it will be rejected by a KiCad 10 *Check Rule Syntax*.

Tier B fires only when a token matches **no** known entry (case-insensitive);
near-misses and anything ambiguous stay silent.

### Tier C — semantic (OUT OF SCOPE)
Value ranges, rule logic, whether a constraint is meaningful on a given layer,
cross-rule interactions — all KiCad's job, never flagged here.

## Severity & control
- Tier A → `Error`, Tier B → `Warning`.
- Setting `kicad-dru.diagnostics`: `"off" | "errors" | "all"` (default `"all"`).
  `errors` shows Tier A only; `off` disables the provider.
- Every diagnostic carries `source: "kicad-dru"` and a stable `code` (e.g.
  `unbalanced-paren`, `unknown-constraint`, `unit-on-unitless`,
  `unknown-property`, `master-only-token`) so a category can be filtered or a
  line suppressed.

## Architecture
Follows the existing pure-function + adapter split (logic in vscode-free `src/`
modules, unit-tested via `node --test`; `src/extension.ts` is the only
vscode-coupled file). Reuses `data/api.json` as the vocabulary source — no new
data needed.

- **`src/parser.ts` (pure).** A small recursive s-expression parser:
  `parse(text) → { ast, errors }`. Tokenises into parens / strings / atoms with
  line+col ranges; builds a tree of list nodes (each with its head atom and
  children); records Tier-A structural errors found during parsing (unbalanced
  parens, unterminated strings). No `vscode` import.
- **`src/diagnostics.ts` (pure).** `computeDiagnostics(text, api, level) →
  Diagnostic[]` where `Diagnostic = { range:{startLine,startCol,endLine,endCol},
  severity:'error'|'warning', code, message }`. Runs the parser, then walks the
  AST: version check, constraint-type + enum + unit-on-unitless checks (Tier A),
  and within condition/assertion string bodies the property/function/layer
  vocabulary checks (Tier B). Honours `level` (off/errors/all). No `vscode`.
- **`src/extension.ts` (adapter).** On activation, create a
  `vscode.DiagnosticCollection('kicad-dru')`. On open + change (debounced
  ~300 ms) of a `kicad-dru` document, call `computeDiagnostics`, map entries to
  `vscode.Diagnostic`, and set the collection; clear on close. Read the
  `kicad-dru.diagnostics` setting and re-validate on its change.
- **`package.json`.** Add `contributes.configuration` for `kicad-dru.diagnostics`.

## Testing
- **Parser unit tests** (`tests/unit/parser.test.mjs`): paren balance, nesting,
  strings with inner quotes, unterminated string, ranges.
- **Diagnostics unit tests** (`tests/unit/diagnostics.test.mjs`): each Tier-A
  and Tier-B check fires on a crafted invalid input with the right range/code;
  the `level` setting gates correctly.
- **Anti-false-positive guard (the load-bearing test):** every existing valid
  fixture under `tests/fixtures/*.kicad_dru` plus the `kicad10-extras` set MUST
  produce **zero** Tier-A diagnostics (and zero Tier-B except the deliberately
  KiCad-11 `kicad11-master-netchain` fixture, which must produce exactly the
  master-only warnings). This guards the governing principle.
- **Invalid fixtures** (`tests/diagnostics/*.kicad_dru` + expected JSON): one per
  check, asserting the produced diagnostics.
- All wired into `npm test` and the CI Test workflow.

## Packaging / docs
- `docs/**` excluded from the `.vsix` (design docs don't ship).
- README "What this extension does not do" updated: diagnostics now partial
  (conservative), KiCad's Check Rule Syntax still the authority.
- CHANGELOG + ROADMAP updated at release.

## Risks & mitigations
- *Catalogue lag* → Tier B is warnings + the `errors`-only setting + the
  master-only token note. Never a hard error on a vocabulary miss.
- *Parser edge cases* (text variables `${}`, escaped chars, comments) → covered
  by parser unit tests; comments stripped per line (line-start `#`, matching the
  grammar) before structural checks.
- *Performance on large files* → debounced, single-pass parser, bounded work;
  no per-keystroke full re-parse beyond the debounce.

## Out of scope (explicit)
Semantic validity, value ranges, rule-logic checks, quick-fixes/code-actions
(possible v0.2.x), and any validation that would require replicating KiCad's
evaluator.
